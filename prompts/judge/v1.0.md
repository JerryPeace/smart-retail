## 評分維度(每維 0.0 - 1.0,獨立判斷)

### 1. Relevance(相關性)— 推薦商品適合這個經銷商嗎?
- 0.0 = 推薦的品類完全不在經銷商過往銷售紀錄中(例:賣手機週邊的店推平板)
- 0.5 = 品類沾邊但定位不對(例:推高階旗艦給以中低階為主的經銷商)
- 1.0 = 商品所屬品類 + 價位帶 + 客群定位都跟過往交易吻合

### 2. Specificity(具體性)— 理由有沒有引用具體證據?
- 0.0 = 純通用形容詞(「市場潛力大」「銷售亮眼」),無數字或事件
- 0.5 = 有部分具體資訊但不齊
- 1.0 = 引用具體數字 + 經銷商過往交易事件 + 商品具體特性,三項齊全

### 3. Actionability(可行動性)— 業務拿著能直接行動嗎?
- 0.0 = 業務看完還要自己查資料才能跟經銷商開口
- 0.5 = 主要 talking point 有了,但缺一兩個關鍵數字(例:沒提建議進貨量)
- 1.0 = 業務可直接打電話/寫 email,不用再查任何東西

### 4. Hallucination(誠實度)— 事實宣稱是否可信?分數高 = 沒有編造
- 0.0 = 推薦理由含明顯捏造(虛構交易紀錄、編造產品規格、引用不存在事件)
- 0.5 = 部分資訊無法從上下文驗證,但不違反明顯事實
- 1.0 = 所有事實宣稱都符合常識或可從上下文合理推得

### 5. Overall(整體品質)— 整體商業價值與可信度
- 0.0 = 業務看完不會用這個推薦
- 0.5 = 業務會用但要大幅修改
- 1.0 = 業務直接照用,經銷商收到後可能下單

**重要**:overall 不是前 4 維機械加權,而是你的整體直覺判斷,捕捉前 4 維沒抓到的東西(語氣專業度、結構合理性、長度恰當性)。可以與前 4 維略有出入。
